# Spec-Driven Playbook

Use this playbook when creating or updating:

- `PRD.md`
- `UI-SPEC.md`
- `TECH-SPEC.md`
- `RFC.md` (major changes)
- `ADR.md` (major decisions)
- `TRACEABILITY.md`
- `ACCEPTANCE-TEST-MAP.md`
- `RELEASE-RISK-CHECKLIST.md`

## Rules

- Keep requirements explicit and testable.
- Assign IDs to requirements (`REQ-###`, `NFR-###`).
- Map every major requirement to implementation and tests.
- Keep docs concise with tables and checklists.
